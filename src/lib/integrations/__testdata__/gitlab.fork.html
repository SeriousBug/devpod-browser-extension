        • Rafał Siejakowski @S-Rafael · 18 hours ago
          Developer

          I know this is just a draft, but maybe we could eliminate these unreadable, one-letter variable names like c, d, s — they are usually not a good idea.

          Secondly, this function does too many things:

          • it computes an affine transformation called tr.
          • it updates the items's transform attributes using that tr and additional translation.
          • it updates the rotation center using the additional translation.

          I suggest splitting it into 3 functions:

          • one function that returns tr
          • another one that can update the rotation center
          • a "composition root" function which calls those two and sets the i2dt_affine of the item.
